Project 2 — Hourly Electricity Load Forecasting

Goal: Build an end-to-end machine learning pipeline to forecast hourly national electricity demand for BEMS decision support.

Overview

Accurate load forecasting is essential for Building Energy Management Systems (BEMS) to optimize HVAC control, energy storage scheduling, and demand response.

This project evaluates how well machine learning models capture hourly and daily electricity demand patterns and demonstrates a full forecasting pipeline.

Example building context where BEMS benefits from accurate load prediction.

Dataset & Key Observation

The dataset contains hourly electricity demand (target) with weather and calendar variables (e.g., temperature, humidity, wind speed, holidays).

  • Target: hourly electricity demand
  • Inputs: weather + calendar + time-series features
  • Split: train / test (time-ordered evaluation recommended)

Exploratory analysis shows temperature alone explains less than half of the demand variation (R² ≈ 0.42), which implies that temporal patterns (lags, hour-of-day, day-of-week) are critical.

Temperature vs load: a single weather feature cannot fully represent daily/weekly demand cycles.

Method

I built a complete pipeline: preprocessing → feature engineering → model training → evaluation.

  • Preprocessing: handle missing values, align timestamps, basic cleaning
  • Feature engineering: lag features (e.g., lag1, lag24), calendar (hour, weekday), rolling stats (optional)
  • Model: Random Forest Regressor
  • Metrics: R², MAE, RMSE
Pipeline summary: time-series features (lags + calendar) improve predictive performance.

Results

The final Random Forest model closely tracked actual electricity demand patterns and achieved strong test performance.

Why Temperature Alone Is Not Enough

Electricity demand is driven by both weather and human activity patterns. Demand can increase in both summer (cooling) and winter (heating), so a single linear temperature relationship is insufficient.

  • Strong hourly and weekly periodicity (workdays vs weekends)
  • Lag features capture “momentum” and routine consumption patterns
  • Temperature helps, but it must be combined with time-series context
Seasonal and periodic patterns explain why time features are essential in load forecasting.

Conclusion & Future Work

This project demonstrates that time-series feature engineering (especially lag features) combined with an ensemble model can significantly improve hourly electricity demand forecasting.

  • Impact: supports BEMS decisions for HVAC operation, storage scheduling, and demand response
  • Next steps: compare Gradient Boosting / XGBoost, add rolling window features, try LSTM/Transformer
  • Validation: use time-based split and evaluate multiple seasons for robustness

Note: Reported metrics depend on dataset split strategy and feature configuration.
