1. Project Overview

This project develops an AI-based Building Energy Management System (BEMS) model that predicts hourly electricity demand in a residential building. Using historical load data and time-based features, three regression models were compared and the best-performing model was selected for integration into a BEMS workflow.

Goal: Build a highly accurate, yet simple and modular forecasting model that can support real-time decision-making in smart home energy management.

2. Motivation

Residential buildings account for a significant share of electricity consumption. However, many households still lack intelligent control of appliances, storage systems, and distributed renewable resources. Accurate short-term demand forecasting is a key requirement for:

  • Reducing electricity bills through peak-shifting and load control,
  • Improving self-consumption of rooftop solar and batteries,
  • Supporting grid stability in regions with high renewable penetration.

3. Methods

We prepared a dataset of hourly electricity demand, along with calendar features such as hour of day, day of week, and whether the day is a weekend or holiday. The data was split into training and test sets to evaluate generalization.

The following regression models were implemented:

  • Linear Regression – simple baseline for comparison,
  • Random Forest Regressor,
  • Gradient Boosting Regressor.

The code was structured as a modular pipeline with separate functions for loading data, preprocessing, feature engineering, model training, and evaluation. This design makes it easy to plug in new models or deploy the pipeline inside a BEMS platform.

Example pipeline step (pseudo-code):
data → preprocess() → make_features() → train_model() → evaluate()

4. Results

Model performance was evaluated using the coefficient of determination (R²) on the test set. Random Forest showed the highest accuracy:

Linear Regression
R² (test) ≈ 0.90
Baseline model with limited ability to capture non-linear patterns.
Gradient Boosting
R² (test) ≈ 0.97
Strong performance but higher training complexity.
Random Forest (Selected)
R² (test) = 0.986
Best trade-off between accuracy, robustness, and implementation simplicity.

Because Random Forest can model complex interactions between features without heavy hyperparameter tuning, it was selected as the final forecasting model.

5. Expected Impact

By integrating the selected Random Forest model into a BEMS environment, the system can:

  • Recommend optimal operation schedules for appliances and storage,
  • Increase the use of on-site renewable energy and reduce grid imports,
  • Provide households with transparent feedback on their demand patterns.

In the long term, this approach can be extended to community-level energy management, supporting regional goals for carbon neutrality.
